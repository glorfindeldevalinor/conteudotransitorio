Perfeito — dá pra amarrar esses campos “pendentes” pegando os metadados (lote, fabricação, validade, palete) do próprio movimento de apanha que você já localiza em MLO_ENDERECOMOVTO (variável vnSeqEnderMovtoApanha). Normalmente, os detalhamentos (por lote) ficam em MLO_ENDERECOMOVTOQTDE. A rotina PKG_FUNCAOLOGISTICA.SP_HISGHISTORDEPOSITO costuma usar exatamente essas infos para escrever o histórico, então vale preencher o máximo possível aqui.

Abaixo te deixo um “drop-in” no ponto onde você já achou vnSeqEnderMovtoApanha, buscando os campos faltantes e usando fallback seguro quando não houver dado. Em seguida, o INSERT em MLOX_LANCTOENDERMOVTO já usa essas variáveis.

O que muda (resumo rápido)

  • Novas variáveis (declare junto com as demais do bloco):
    vdDtaFabricacao DATE; vdDtaValidade DATE; vnNroLoteFab NUMBER; vnSeqPaleteRF NUMBER;

  • Busca dos metadados do lote com base em vnSeqEnderMovtoApanha:

    • Fonte primária: MLO_ENDERECOMOVTOQTDE (por ser nível “por-lote”).

    • Fallback: deixa NULL/0 se não achar.

  • INSERT preenchendo:

    • DTAFABRICACAO, DTAVALIDADE e NROLOTE vindos do select acima (sem forçar SYSDATE).

    • STATUSENDERORIGEM, INDENTRADASAIDAORI, LOCALCANCEL: mantém os fixos que você já usa ('O', 'E', 'A').

2) Depois que você encontra vnSeqEnderMovtoApanha, busque os dados do lote

Substitua/complete logo após este trecho seu (que já existe):

plsql
-- Buscar vnSeqEnderMovtoApanha (conforme log)
BEGIN
  SELECT MIN( SEQENDERECOMOVTO ) 
    INTO vnSeqEnderMovtoApanha
    FROM MLO_ENDERECOMOVTO A
   WHERE A.NROCARGA       = vnNroCarga
     AND A.CODDEPOSSEPAR  = vsCodDeposito
     AND A.SEQLOTE        = vnSeqLote
     AND A.NROQUEBRA      = vnNroQuebra
     AND A.SEQPRODUTO     = vnSeqProduto
     AND A.CODDEPOSITANTE = vnCodDepositante
     AND A.TIPMOVTO       = 'E'
     AND A.INDENTRADASAIDA = 'S'
     AND Exists( Select * from MLO_ENDERECOMOVTOQTDE B
                 Where A.SEQENDERECOMOVTO = B.SEQENDERECOMOVTO );
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    vnSeqEnderMovtoApanha := NULL; 
  WHEN OTHERS THEN
    vsMsgErroLoop := vsMsgErroLoop || 'Produto ' || vnSeqProduto || ': Erro ao buscar SEQENDERECOMOVTO para SP_HISGHISTORDEPOSITO: ' || SUBSTR(SQLERRM, 1, 200) || '. ';
    vnProdutosComErro := vnProdutosComErro + 1;
END;

Cole em seguida este bloco para capturar fabricação/validade/lote/palete:

plsql
-- Captura metadados do lote/palete a partir do movimento de apanha
vdDtaFabricacao := NULL;
vdDtaValidade   := NULL;
vnNroLoteFab    := NULL;
vnSeqPaleteRF   := NULL;

IF vnSeqEnderMovtoApanha IS NOT NULL THEN
  BEGIN
    /* Fonte preferencial: tabela de quantidade por movimento (nível de lote) */
    SELECT 
           /* se houver múltiplas linhas, pega um agregado “estável” */
           MIN(q.DTAFABRICACAO) AS DTAFABRICACAO,
           MIN(q.DTAVALIDADE)   AS DTAVALIDADE,
           MAX(q.NROLOTE)       AS NROLOTE,
           MAX(NVL(q.SEQPALETERF, 0)) AS SEQPALETERF
      INTO vdDtaFabricacao, vdDtaValidade, vnNroLoteFab, vnSeqPaleteRF
      FROM MLO_ENDERECOMOVTOQTDE q
     WHERE q.SEQENDERECOMOVTO = vnSeqEnderMovtoApanha;

    /* Se a QTDE não tiver palete, tenta da própria cabeceira do movimento */
    IF NVL(vnSeqPaleteRF,0) = 0 THEN
      BEGIN
        SELECT MAX(NVL(m.SEQPALETERF,0))
          INTO vnSeqPaleteRF
          FROM MLO_ENDERECOMOVTO m
         WHERE m.SEQENDERECOMOVTO = vnSeqEnderMovtoApanha;
      EXCEPTION
        WHEN OTHERS THEN
          NULL; -- mantém nulo/0
      END;
    END IF;

  EXCEPTION
    WHEN NO_DATA_FOUND THEN
      NULL; -- deixa nulo, cai nos defaults do INSERT
    WHEN OTHERS THEN
      vsMsgErroLoop := vsMsgErroLoop || 'Produto '||vnSeqProduto||
                       ': Erro ao buscar lote/fabricacao/validade/palete do movimento de apanha: '||
                       SUBSTR(SQLERRM,1,200)||'. ';
      -- segue com valores nulos/zero
  END;
END IF;

Observação: se a sua NROLOTE for VARCHAR2, troque o tipo da variável e remova NVL(…,0) onde aplicável.
Se sua modelagem guardar fabricação/validade/nroLote em outra tabela (ex.: MLO_ENDERECOLOTE ou um histórico), dá pra repetir a mesma estratégia: tentar MLO_ENDERECOMOVTOQTDE primeiro e, se vier nulo, fazer um SELECT auxiliar nessa outra fonte.

Notas rápidas sobre cada campo

  • DTARECEBIMENTO: mantenha SYSDATE (timestamp da geração do estorno).

  • DTAFABRICACAO / DTAVALIDADE / NROLOTE: melhor fonte é MLO_ENDERECOMOVTOQTDE do vnSeqEnderMovtoApanha. Se não houver, deixe NULL (ou tente uma segunda fonte, p.ex. o “lote” corrente do endereço).

  • SEQPALETERF: pode vir de MLO_ENDERECOMOVTOQTDE.SEQPALETERF; se não tiver, tente MLO_ENDERECOMOVTO.SEQPALETERF; se nada, 0.

  • STATUSENDERORIGEM / INDENTRADASAIDAORI / LOCALCANCEL: você já usa 'O', 'E', 'A'. Esses valores fazem sentido como: endereço Origem, E de “saída” no lançamento de estorno e local A (cancelado/ajuste) — mantenha se isso está alinhado com seu legado.

  • CODUSUARIO: seu fixo 'CONSINCO' tá ok (ou substitua pelo usuário real do integrador se precisar trilha).
